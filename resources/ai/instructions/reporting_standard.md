# Reporting Standard

Reports should include:

- User issue.
- System context.
- Evidence collected.
- Findings.
- Actions taken.
- Commands/tools used.
- Artifacts and citations.
- Verification results.
- Cleanup performed.
- Residual risk and follow-up.

Use a technician-readable handoff style: short sections, clear bullets, exact
paths for artifacts/logs, and plain-language conclusions. When the app
generates session reports, preserve both the HTML handoff copy and markdown
source.

Avoid secrets, API keys, passwords, and unnecessary personal data.
